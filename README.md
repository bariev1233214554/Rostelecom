Тестирование страницы авторизации Ростелекома - https://b2c.passport.rt.ru

Перед запуском тестов установите зависимости командой: pip install -r requirements.txt

1 conftest.py - Набор фикстур для проведения тестов
    -Инициализация и закрытие браузера
    -Словарь со всеми учетными данными для тестов из фаила .evn
    -автоскриншот при падении теста.

2 .env - Набор данных для проведения тестирования: email, телефон, пароль, счет, логин, имя, фамилия.
Данные как для позитивных, так и негативных сценариев автотестов.

3 Папака screenshots - скриншоты провалов автотестов.

4 Папака test - Набор автотестов для тестирования сайта: тесты берут данные из фаила .eve.
        Изменяя данные в файле .eve (есть готовые шаблоны) можно провести как позитивные, так и негативные тесты.

    4.1. (test_avtorization_chet) Тест для проверки авторизации по лицевому счету. 
    
    4.2 (test_avtorization_email) Тест для проверки авторизации по email.        

    4.3 (test_avtorization_login) Тест для проверки авторизации по Логину

    4.4 (test_avtorization_mail) Тест для проверки авторизации по кнопки mail

    4.5 (test_avtorization_ok) Тест для проверки авторизации по кнопки Однокласники
    
    4.6 (test_avtorization_phone) Тест для проверки авторизации по телефону

    4.7 (test_avtorization_tinkoff) Тест для проверки авторизации по кнопки Тинькофф. 
    Для теста нужно мобильное устройство с сим-картой к номеру которой привязан аккаунт Тинькофф

    4.8 (test_avtorization_vk) Тест для проверки авторизации по кнопки VK.
    Для теста нужно устройство с установленным приложением Vk. Сканер vk qr должен быть открыт.

    4.9 (test_avtorization_yandex) Тест для проверки авторизации по кнопки Яндекс.
    Для теста должен имется доступ к Яндекс id для получения и введения кода.

    4.10 (test_buttons_confidens) Тест для проверки кнопки Политикой конфиденциальности в подвале сайта.

    4.11 (test_buttons_contact) Тест для проверки кнопки связи в подвале сайта.

    4.12 (test_buttons_cookes) Тест для проверки кнопки Cookies в подвале сайта.

    4.13 (test_buttons_help) Тест для проверки кнопки помощи.

    4.14 (test_buttons_obrabotka_person_dann) Тест для проверки кнопки Политикой обработки персональных данных.

    4.15 (test_buttons_polz_soglas) Тест для проверки кнопки Пользовательского соглашения в подвале сайта.

    4.16 (test_buttons_soglasie) Тест для проверки кнопки c Пользовательским соглашением.

    4.17 (test_registration) Тест для проверки регистрации. 
    Для теста нужен ручной ввод кода подтверждения с телефона / email.

    4.18 (test_tabulation) Тест для проверки таба выбора аутентификации. Проверяет сразу все типы аутентификации.

    4.19 (test_tabulation_passrec) Тест для проверки таба выбора аутентификации  при восстановлении пароля. Проверяет сразу все типы аутентификации.

    4.20 (test_vostanovlenie_password_chet) Тест для проверки восстановления пароля по счету. Тест успешен в случае авторизации по новому паролю.
    Для теста нужен ручной ввод кода подтверждения с телефона / email. 
    Выбор метода восстановления зависит от значения параметра TEST_PHONE в файле .env если телефон пустой - email.

    4.21 (test_vostanovlenie_password_email) Тест для проверки восстановления пароля по email. Тест успешен в случае авторизации по новому паролю.
    Для теста нужен ручной ввод кода подтверждения с телефона. 
    Выбора метода восстановления в этом тесте нет - восстанавливает по email

    4.20 (test_vostanovlenie_password_login) Тест для проверки восстановления пароля по логину. Тест успешен в случае авторизации по новому паролю.
    Для теста нужен ручной ввод кода подтверждения с телефона / email. 
    Выбор метода восстановления зависит от значения параметра TEST_PHONE в файле .env если телефон пустой - email.

    4.21 (test_vostanovlenie_password_phone) Тест для проверки восстановления пароля по телефону. Тест успешен в случае авторизации по новому паролю.
    Для теста нужен ручной ввод кода подтверждения с телефона. 
    Выбора метода восстановления в этом тесте нет - восстанавливает по телефону.
